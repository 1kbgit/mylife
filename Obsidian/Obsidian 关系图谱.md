- **点（Node）**：每一个圆点，对应你仓库里的**一个 Markdown 文件**（比如你现在的「倒立」「心情」「2026.4.15 以及之前」，就是 3 个独立的 md 文件）
- **线（Link）**：两个点之间的连线，代表两个文件之间有**双向链接 / 引用关系**
    
    - 比如你在「心情.md」里写了`[[倒立]]`，就会在两个点之间生成一条线，代表它们有关联
    
- **点的大小**：点越大，代表这个文件被其他文件引用的次数越多（越核心、越重要）
- **点的颜色**：可以自定义，用来区分不同分类（比如日记、目标、工作）
- ### 1. 基础交互（先玩明白界面）

- **平移 / 缩放**：鼠标滚轮缩放，按住左键拖动图谱，调整视角
- **聚焦单个文件**：直接点击圆点，会高亮这个文件 + 所有和它关联的文件，其他文件会变暗，快速看这个文件的关联网络
- **打开文件**：双击圆点，直接打开对应的 md 文件，编辑内容
- **搜索定位**：右上角搜索框，输入文件名 / 关键词，直接定位到对应圆点，快速找文件

### 2. 核心：给文件加链接，生成连线（让图谱「活」起来）

这是图谱的核心用法，用 Obsidian 的**双向链接**语法，给文件建立关联：

1. 打开任意一个 md 文件（比如「心情.md」）
2. 在内容里输入 `[[`，会自动弹出文件列表，选择你要关联的文件（比如「倒立.md」）
3. 输入完成后，就会生成 `[[倒立]]` 这个双向链接
4. 保存文件后，回到关系图谱，就能看到「心情」和「倒立」两个点之间，出现了一条连线！
 举个你日记的实用例子：

在「2026-04-16 日记.md」里写：

markdown

```
# 2026-04-16 日记
## 今日心情
今天状态不错，继续坚持[[倒立]]训练，[[心情]]也很稳定。
```

保存后，「2026-04-16 日记」这个点，就会同时和「倒立」「心情」两个点连线，你的日记就变成了一张完整的知识网络。

## 四、进阶优化：让图谱更好用、更美观

### 1. 自定义图谱设置（右上角齿轮图标）

- **显示设置**：
    
    - 「显示文件名称」：默认开启，方便识别每个点
    - 「显示附件 / 附件文件夹」：关闭，避免图谱里出现无关的图片 / 文件夹
    - 「深度限制」：新手先设为 2，避免图谱太乱，后续再调整
    
- **样式设置**：
    
    - 「节点大小」：按「引用次数」调整，核心文件自动变大
    - 「节点颜色」：按「文件夹 / 标签」分组，给「日记」「小目标」设置不同颜色，一眼区分分类
    - 「力导向」：调整节点的疏密，让图谱更舒展
    
- **过滤设置**：
    
    - 可以按「文件夹」「标签」过滤，比如只看「小目标」文件夹的内容，专注看倒立相关的记录
    

### 2. 用标签给图谱上色（更直观的分类）

- 在文件里加标签，比如「倒立.md」里加`#小目标/倒立`，「心情.md」里加`#日记/心情`
- 在图谱设置里，开启「按标签着色」，不同标签的点会显示不同颜色，分类更清晰
- 可以在过滤里，只显示某个标签的内容，比如只看`#小目标`的所有文件，专注看目标进度

### 3. 本地图谱（Local Graph）：聚焦单个文件

- 打开任意文件，点击左侧边栏的「关系图谱」图标（两个点连线的图标），打开**本地图谱**
- 本地图谱只会显示「当前文件 + 所有和它关联的文件」，比如打开「倒立.md」，只会显示倒立相关的所有日记、子文件，不会被其他内容干扰，适合复盘单个主题

## 五、避坑 & 最佳实践

1. **不要乱加链接**：只加真正有逻辑关联的链接，比如日记关联主题，不要无意义的互链，否则图谱会变成一团乱麻
2. **用文件夹 + 标签双重分类**：文件夹做一级分类（日记 / 小目标），标签做二级分类（# 倒立 #心情），图谱会更有条理
3. **定期整理图谱**：每周花 5 分钟，给日记补全链接，清理无效链接，让图谱保持整洁
4. **不要用图谱替代文件管理**：图谱是「可视化工具」，不是「文件管理器」，核心还是靠文件夹 + 文件来管理内容，图谱是辅助你梳理关系的
