# Markdown 使用手册
#Markdow[[Markdown使用教程]]
Markdown 是纯文本格式，**完全兼容 Git 和 Obsidian**，不用记复杂语法，只需要掌握下面这些，就能覆盖你 99% 的日记、打卡、记录需求，写出来的内容既整洁又有格式，还能完美保留所有修改历史。

---

## 一、核心语法（按日记高频使用排序，直接复制就能用）

### 1. 标题（日记分级必备）

用 `#` 加空格，`#` 越多标题层级越小，Obsidian 会自动生成目录，方便你快速跳转。

markdown

```
# 一级标题（日记大标题，比如「2026-04-16 日记」）
## 二级标题（模块标题，比如「今日心情」「倒立打卡」）
### 三级标题（子模块，比如「训练细节」「情绪复盘」）
```

✅ 效果：

# 2026-04-16 日记

## 今日心情

### 情绪复盘

---

### 2. 文字样式（突出重点，比纯 txt 好用太多）

markdown
#加粗 **加粗**

```

**加粗内容**：用两个星号包裹，标记重点事件、核心情绪
*斜体内容*：用一个星号包裹，标注补充说明、内心想法
~~删除线内容~~：用两个波浪线包裹，标记作废的计划、已完成的事项
`行内代码`：用反引号包裹，记录命令、打卡数据（比如倒立时长）
```

✅ 效果：

**今天倒立突破 30 秒！**

_今天状态不错，比昨天进步明显_

~~原计划练 10 组，实际完成 8 组~~

`倒立时长：35秒`

## 二、适配你「日记 + 倒立打卡」的完整模板（直接复制用）

markdown

```
# {{date:YYYY-MM-DD}} 日记
## 今日心情
**今日状态：** 稳定/波动/亢奋
> 今日感悟：
> 坚持倒立的第X天，核心力量持续提升，继续保持！

## 倒立打卡
- [x] 热身5分钟
- [x] 靠墙倒立练习5组，最长时长`35秒`
- [x] 放松拉伸5分钟

### 训练数据
| 项目       | 今日数据 | 目标数据 |
| ---------- | -------- | -------- |
| 最长时长   | 35秒     | 60秒     |
| 训练组数   | 5组      | 6组      |
| 状态评分   | 9分      | 10分     |

## 关联记录
[[倒立]]
[[心情]]
[[2026-04-15 日记]]

#日记 #倒立打卡 #心情记录
```

> 说明：`{{date:YYYY-MM-DD}}` 是 Obsidian 模板语法，新建日记会自动填充当天日期，不用手动改。

## 三、Obsidian 里的 Markdown 实用技巧

1. **实时预览**：按 `Ctrl+E` 一键切换「编辑模式 / 预览模式」，写的时候看代码，写完看最终效果，和 Word 一样直观。
2. **分屏预览**：右键标签页 →「向右拆分」，一边写代码一边看渲染效果，不用来回切换。
3. **自动补全**：输入 `#` `*` `[[` 时，Obsidian 会自动补全语法，不用手动记。
4. **格式刷**：选中内容，点击顶部工具栏的对应按钮（加粗、斜体、列表），一键生成格式，新手零门槛。
5. **导出为 Word/PDF**：Obsidian 支持直接把 Markdown 文件导出为 Word、PDF，完美保留所有格式，满足你之前的格式需求。

## 四、避坑指南（新手必看）

1. **语法符号和内容之间必须加空格**：比如 `# 标题`，不能写成 `#标题`，否则不会生效。
2. **列表、引用块必须单独占一行**：不要和其他内容写在同一行，否则格式错乱。
3. **不要用复杂格式**：日记只需要用上面的基础语法就够了，越简单越稳定，Git 兼容性越好。
4. **图片不要用绝对路径**：Obsidian 自动生成的相对路径最稳妥，不要手动改路径，避免图片丢失。
5. **特殊字符转义**：如果要显示 `*` `#` 等语法符号，前面加 `\` 转义，比如 `\#` 会显示 `#`。
